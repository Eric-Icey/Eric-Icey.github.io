操作系统

系统灵魂 核心专业课 底层开发基础

一 操作系统概述

(一). 初步认识:

  1. 没有操作系统依旧可以正常使用程序
    • 只能使用刻录在计算机硬件上的bios程序
    • 操作系统与普通软件有重大差别
    • 例如在蓝屏界面设置计算机基本信息时
    • 此类程序的编写与安装都需要特殊的方式
  2. 普通用户无法运行正常软件
  3. 操作系统提供的功能:
    1. 可视化操作界面;
    2. 控制普通程序运行；
    3. 控制多个程序并发运行，管理系统资源；
    4. 进行系统设置/配置，监控系统状态；
    5. 提供工具软件集合；
    6. 信息的存取及其保护机制；
  4. 操作系统对应用程序运行的支持：
    1. 存放.exe文件；
      • 提供对应的接口，将文件放在特定的硬盘目录下；
    2. 启动.exe文件；
    3. 为程序分配内存；
      • 待解答;
    4. 为程序提供输入/输出的驱动/硬件接口；
    5. 如何处理程序死循环对CPU的影响？
      • 由于分时系统的存在，该循环程序只会在获得CPU资源时有效，其余时间内其它程序仍能正常运行
    6. 程序如何退出系统？
      • 待解答;
  5. 课后问题
    1. 没有安装操作系统的计算机启动过程和结果？
      • 启动：启动后直接显示为DOS或者时BIOS界面
      • 过程：使用简单的键盘进行计算机基本设置的配置，比如修改参数一类
      • 结果：仅能修改计算机本身的属性，但无法进行其他更为高级的设置
    2. 常见的虚拟机软件(VMware VirtualPC)能不能理解为操作系统？
      • 不能。
      • 虚拟机软件只是为操作系统提供了一个可以安装的地方，其本质应该也是属于普通计算机的范畴，其所有的硬件设置都是虚拟出来的，拥有其对应的参数。如需正常使用仍需额外安装操作系统。

(二)操作系统功能和定义

  • 功能:
    1. *进程管理（CPU管理）：
      多个程序运行时，Windows环境下有多个进程在并发运行->共享CPU
      1. 进程控制：创建，暂停，唤醒，撤销；
      2. 进程调度：调度策略，优先级
      3. 进程间通信；
    2. *内存管理：
      1. 内存分配：
        • 内存小也可运行程序，但会较为卡顿，因为每次只能运行其所需要的一部分
      2. 内存共享；
      3. 内存保护；
      4. 控制虚拟内存；
    3. 设备管理(第八章)：
      1. 设备分配/调度；
        • 通过对应的接口管理各类外设的操作
      2. 设备无关性；
      3. 设备传输控制：
        • 设备与CPU/内存设备；
      4. 设备驱动；
    4. 文件管理：
      1. 存储空间管理；
      2. 文件操作：
        • 创建，删除，复制，粘贴等；
      3. 目录操作；
      4. 文件/目录的存取管理权限；
        • 软件需要满足进程管理/内存管理才能称之为操作系统
    • 通用/实用的操作系统可能会有更多的功能：
      • 分布式管理，图形资源处理等;
  • 定义：
    • 大型系统程序：
      1. 提供用户接口；
      2. 为应用程序分配/调度软硬件资源，控制/协调应用程序的并发活动；存取和保护信息
  • 地位：
    • 处于应用程序（软件）与硬件系统中间，承上启下；
  • 课后习题
    • 现代操作系统会不会因为内存太小而让应用程序无法启动运行或让系统崩溃？
      • 不会
      • 操作系统具有内存管理的能力，应用程序即时使用的内存是由一定的限制的，操作系统对其实时内存进行一个动态分配管理使其能正常运行，但由于内存总量过少的原因，调度方面可能会不太良好从而导致应用程序在运行时会有卡顿的现象出现。
    • 如何理解操作系统两大功能：扩展的虚拟机和资源管理？
      • 扩展的虚拟机：
      • 资源管理：资源管理应包含多个方面，包括进程~，内存~，文件~,设备~等，帮助使用者更好的控制计算机，并且为应用程序的并发运行提供一定的保障

(三)操作系统发展历史

  • 发展两大动力：用户需求提升&硬件技术进步
  • 现代的大多数应用都需要非常高效实用的硬件支持，如深度学习，图像处理，大数据计算等
  • 计算机硬件的四个典型阶段：
    1. 电子管时代：1946-1955；
    2. 晶体管~： 1955-1965；
    3. 集电~：1965-1980；
    4. 大规模集电：1980-至今；
      ——速度，容量，稳定性，可靠性不断提升
  • 操作系统的四个阶段：
    1. 手工操作:50s e;
    2. 单道批处理:50s;
    3. 多道批处理:60s e;
    4. 分时系统:60s m;
  • 手工操作-电子管时代(没有操作系统)：
    1. 结构：
      • 硬件：电子管，接线面板(按钮，开关，接线柱等)；
      • 程序：二进制程序；打孔：纸带/卡片；
    2. 使用：
      • 程序准备。启动。结束：手工处理，繁琐耗时；
    3. 缺点：
      • 效率极低 用户独占 缺少交互；
  • 单道批处理-晶体管时代：
    • 工作特点：
      1. 批量：多个作业输入磁盘形成作业队列；
      2. 自动：一次处理队列中每个作业：
        • 装入-运行-输出(循环处理)
        • 程序的装入和撤出：自动完成，效率高
      3. 单道：依次，串行；
        • CPU利用情况：
      • CPU和外设交替空闲和忙碌，CPU和外设利用效率都很低
  • 多道批处理：
    • 定义：内存中存放多道程序，当一个程序不再使用CPU时，调用另一个程序来占用CPU相互穿插运行；
    • 特点：
      1. 多道：内存上同时存放多道程序；
      2. 宏观上并行，微观上串行；
    • 缺点：
      1. 作业处理时间长；
      2. 交互能力差；
      3. 运行过程不确定：无法确定当前运行的具体程序；
  • 课后习题
    • 再多到批处理中尝试不断增加程序的数量，系统的而效率会不会持续增加？结果会如何？
      • 不会
      • 多道批处理能处理的程序数量是有峰值的，未到峰值之前数量越多越好，超过峰值之后由于其本质还是在CPU上串行处理便有可能导致部分程序无法准时利用CPU而降低实用效率

(四)分时技术&分时操作系统

  • 60s硬件的重大进展：
    • 中断技术：
      • CPU收到外来中断信号后，停止当前工作，转而处理中断信号发出事件，处理完后回到中断点继续原有工作；
    • 通道技术：处理外设与内存之间的数据传输；
  • 分时操作系统背景：
    1. 事务性任务的涌现：
      • 交互性高，快速响应；
    2. 要求支持多任务/多用户；
    3. 出现多终端计算机：
      • 高性能主机—+多个终端：
        • 主机：运算，CPU+内存；
        • 终端：输入&显示；
      • 主机采用分时技术轮流为每个终端服务；
  • 分时技术：
    1. 主机以”时间片”为单位，把CPU轮流分配给每个终端使用，直到全部作业被运行完毕；
    2. 时间片极短，在终端数量不多时，每个终端都能很快重新获得CPU的使用从而得到及时的响应
      等待周期 = 时间片 * 终端数量
  • 分时系统特点：
    • 多路调制性 | 独占性 | 交互性
  • 大型分时系统的实践：MUltics
  • 第一个实用化的分时操作系统:UNIX:
    • 体现操作系统领域各种先进概念和技术
    • 革新和创造：
      1. 操作系统的可移植性；
      2. 硬件无关性；
      3. 将外设当成特殊的文件类型进行处理，进行统一管理；
    • 现主要用于服务器而非个人用户
  • 分时技术的时间片设置太大或太小，应该根据哪些因素来考虑？是尽量大好还是尽量小好？
    • 根据需要使用到CPU的所有终端的数目来考虑，同时还需要考虑到硬件的性能支持
    • 其大小应该研究该项操作系统所可能使用的程序/进程后，选出一个合适的区间，避免过大而无法计时交互，过小又无法充分发挥CPU性能的问题。

(五)典型操作系统类型

  • 操作系统的进一步发展（分时系统的演化)
    1. 微机操作系统：
      • 背景：大规模集电的发展->个人计算机时代
      • 1973：CP/M操作系统
        • 良好的层次结构：BIOS把操作系统和硬件分离
      • 1976：苹果Mac Os：
        1. 运行在Macintosh系列电脑；
        2. 首个配有图形界面和鼠标
      • 80s ：微软MS DOS
        1. 磁盘操作系统；
        2. 单用户
        3. 预装在IBM PC上
      • 微软Windows;
      • 对比手机嵌入式
        • 更高的硬件配置
        • 更高的存取能力
    2. 实时操作系统：
      • 背景：
        1. 实时事务：军事，工业控制，智能仪器；
        2. 要求：某些任务优先紧急处理
      • 特点：强调任务完成时限
        1. 硬实时系统：要求非常严苛
        2. 软实时系统：要求不严格，但尽量快
          • 直播，网游，网络视频等
    3. 嵌入式操作系统(涵盖范围比实时操作系统更大)：
      • 软硬件可裁剪，软硬件一体化的系统
      • 典型嵌入式操作系统：
        • Andriod/Linux(非PCLinux)/usOS/ucLinux/vxWorks/WinCE
    4. 网络操作系统：
      • 普通操作系统+网络通信+网络服务
      • 功能：网络资源透明存取&存取控制
    5. 开源操作系统：
      • Minix Os(小型)：
        • 特点：
          1. 开放源码；
          2. 12,000行
          3. 类UNIX系统
        • 官网
        • 缺点：
          1. 缺少多线程
          2. 内核本身不需要过度的可移植性
      • Linux (大型)：
        • 最早版本：Linux 0.1
        • Linux源码地址
        • 考虑了过多的可移植性
        • 商用领域用的非常多，路由器，服务器等
        • 缺点：
          1. 宏内核在整体设计有害
          2. 架构的耦合度太高
    6. 多处理机操作系统
  • 你认为国产操作系统能否成功？影响成功的因素有哪些？
    • 在某些特定情况下的国产操作系统是能成功的，如军事管理方，这样的小众且与国家密切相关的地方实行国产操作系统也能够根据我国状况进行适应性的调整，随着硬件的不断提升，一定能成功。但是对于广大普通民众，国产操作系统的接受率应该会非常低，因为现在市面上流行的操作系统已经发展很多年，各方各面都很完善，大众也已经习惯了这样的操作系统，如果国产不能够优于这些流行操作系统过多的话，鲜有人会放弃熟悉的去使用。由于产品垄断的原因，导致国产开发出优于市面流行系统更为艰难，所以民众化的普及是很难成功的。

章节测试

问答

  1. 站在普通用户的角度，总结操作系统有哪些基本功能？
    1. 提供图形化界面方便操作
    2. 提供应用程序的控制，使普通用户可以更好的使用各类程序而不用了解其内部构造
    3. 提供计算机资源的管理，使普通用户能够方便的存取资料，信息等电子信息
    4. 提供普通的系统工具集合，帮助用户高效的完成一些简单但人力耗时长的任务，如计算器等
  2. 操作系统有哪4个典型的发展阶段，各有什么特点？
    1. 手工操作阶段
      • 使用接线面板和电子管作为硬件资源
      • 程序员直接编写二进制程序，并使用打孔的纸带/卡片进行程序的录入
      • 效率极其低下，且无法有效交互，资源无法共享只能单独使用
    2. 单道批处理阶段
      • 使用晶体管作为硬件资源
      • 将所需完成作业排成队列一次性输入系统进行运算
      • 可以自动完成多项作业，但对CPU和外设利用效率低下，同样无法有效交互和共享
      • 单次循环作业处理效率高
    3. 多道批处理阶段
      • 系统同时可以接收多道程序进行运算
      • 交替运行程序占用CPU，提高外设和CPU利用效率
      • 程序运行过多无法准确判断当前所运行的程序是哪个，并且大量程序交替运行也使得总的作业处理时间增长
      • 交互能力较差
    4. 分时操作阶段
      • 以中断技术为核心
      • 使用时间片作为计时单位，提高CPU利用效率
      • 良好的交互性，同时对外来作业处理的响应速度也极快
      • 每项作业在使用时独占CPU增加其运行速率
      • 能够同时为大量的终端服务
  3. 多道批处理系统为什么工作效率比单道的高？
    • 多道可以同时储存更多的作业在队列中，同时使用多程序交替使用CPU和外设的架构，缩短了等待时间，从而提升工作效率
  4. 分时技术与多道批处理都能完成多个程序的切换。这两种切换情形有什么差别？
    • 交互性：分时技术可以临时为外来作业优先处理
    • CPU利用效率：分时技术使用时间更短的时间片为各大进程提供CPU资源，多道对每项作业的处理时间不一致，会有程序闲置或CPU闲置情况出现
  5. 从系统观点来看，操作系统的核心功能包括哪些？
    1. 进程(CPU)管理
      1. 进程控制
      2. 进程间通信
      3. 进程调度
    2. 内存管理
      1. 内存分配 管理
      2. 内存资源保护
      3. 虚拟内存
    3. 文件管理
      1. 存储空间
      2. 文件基本操作
      3. 文件目录操作
    4. 设备管理
      1. 外部设备与应用程序的接口
      2. 设备的分配/调度

二 操作系统逻辑结构

(一)操作系统逻辑结构

  • 逻辑结构：
    • OS的设计和实现思路；
  • ~种类：
    1. 整体式结构；
      1. 以模块为基本单位构建；
        • 如同GO，C语言之类的‘
      2. 特点：
        1. 模块设计，编码，调试独立；
        2. ~调用自由；
        3. ~通信多以全局变量完成
          • 难以控制变量
      3. 缺点：
        • 信息传递随意，维护和更新困难；
    2. 层次式~；
      1. 软件例子：TCP/IP协议栈
      2. 往下结合硬件，向上结合用户需求
      3. 功能模块按照调用次序排列，相邻层间只有单向依赖|单向调用
      4. 分层原则：
        • 硬件相关——最底层
        • 外部特性——最外层
        • 中间层——调用次序|消息传递顺序
        • 共性的服务——较低层
        • 活跃功能——较低层
      5. 优点：
        • 结构清晰，避免循环调用
        • 整体问题局部化系统的正确性容易保证
        • 有利于操作系统的维护，扩充和移植
    3. 微内核~：
      • 操作系统 = 微内核 + 核外服务器
        • 微内核：
          • 足够小，提供OS最基本的核心功能+服务
          • 实现与硬件精密相关的处理
          • 实现较为基本的功能
          • 负责客户和服务期间的通信
        • 核外服务器：
          • 完成OS的绝大部分服务功能，等待应用程序提出要求
          • 若干服务器/进程共同构成
            • 进程/线程服务器 虚存服务器 设备管理服务器，以进程形式运行在用户态
      • 客户服务器结构；
  • OS的微内核结构的优点，对设计程序有什么启示？

(二)CPU的态

  • OS的最基本硬件：CPU
    • CPU:
      • CPU态(Mode)
        1. CPU的工作状态
        2. 对资源和指令使用权限的描述
        3. 特权指令：和底层命令相关，CPU在高权限状态下才可使用
          ![image.png](https://i.loli.net/2020/02/27/wH68ugK5M4DClEi.png)
      • 态的分类：
        1. 核态
          • 访问全部资源+执行所有指令
          • 管理程序/OS内核
        2. 用户态
          • 资源访问受限
          • 用户程序
        3. 管态
          • 介于上述两者之间
        4. 用户/核态的转换，为什么需要转换？
          • 用户->核
            • 请求服务
            • 发生中断
            • 进程错误
            • 用户企图执行特权指令
          • 核->用户
            • 执行中断返回
      • 硬件和OS对CPU的观察
        • 硬件按照 态 区分CPU状态
        • OS按照 进程
        • 不同类型CPU的态的定义是不同的
      • Intel CPU的态：
        • 待解答
      • 存储器:存储程序和数据的部件
        • 分类
          • 存储器读写方式：
            • RAM & ROM
          • 按存储元的材料
            • 主存：半导体~
            • 磁盘：磁存储器
            • 光盘：光存储器
          • 按和CPU的联系
            • 主存：直接交换信息，与CPU三种总线直接相连
            • 辅存：相反
          • 存储体系
            • 理想：速度快，容量大，成本低
            • 实际：
              • 寄存器
              • 高速缓存(CACHE)
              • 主存
              • 缓存
            • 工作原理
              • CPU读取指令|数据的访问顺序
                1. 缓存(命中)
                2. 内存(没有命中)
                3. 辅存(缺页)

(三)中断机制

  1. 定义：
    • CPU对突发的外部事件的反应过程|机制
    • 引入中断的目的
      1. 实现并发
      2. 实时处理
      3. 自动处理故障
  2. 一些概念：
    • 中断源&中断类型
      • 源：引起中断的事件
      • 类型：
        1. 强迫性&自愿
          • 强迫性：程序没有预期，如外部中断
          • 自愿：执行访管指令？
        2. 外中断&内中断
          • 外中断类型：不可屏蔽&可屏蔽
            1. 紧要,CPU必须响应
            2. 不紧要，CPU可以不用响应
      • 断点：
        • 程序中断的地方，将要执行的下一指令的地址
        • CS:IP
      • 现场
        • 程序正确运行所依赖的信息集合
      • 现场的两个处理过程
        • 保护：进入中断服务之前，进栈保护
        • 回复：退出~后，出栈恢复
      • 中断响应过程
        1. 识别中断源
        2. 保护断点和现场
        3. 装入中断服务程序的入口地址(CS:IP)
        4. 进入中断服务程序
        5. 恢复现场和断点
        6. 中断返回IRET
      • ~实质：
        • 交换指令执行地址
        • 交换CPU的态
        • 工作：
          • 现场的保护和恢复
          • 参数传递(通信)

第二章章末

  1. 操作系统微内核架构有何作用？在应用编程中，我们如何使用“客户-服务器”的编程思想提升程序的功能可扩展性？
    1. 提高系统的可扩展性和可移植性
    2. 增强了系统的可靠性
    3. 提供了对分布式系统的支持
    4. 融入了面向对象技术
    5. 程序只负责最核心的逻辑业务，同时提供许多接口，使更多的普遍性工作交由通过接口连接的外部程序负责，降低耦合，有利于维护和增加功能。更换外部程序或者环境就几乎不会对程序本身产生影响，同时对外接口众多也能满足不同的需求。
  2. 下载Linux 0.11 OS源代码，用SourceInsight软件打开源代码，了解其源代码的构成，大致的模块结构,做简单的文字描述。
  3. 网上搜索“Intel CPU的保护模式”，“特权级”两个关键词，了解“CPU的态”与“保护模式”、“特权级”等概念。
    1. CPU的态主要是为了区分不同的使用”用户”以保证其操作不会对OS本身造成损害
    2. 保护模式采用了更为广阔的内存映射方式，扩大了实模式下能使用的内存上限。同时保护模式还提供了不同程序(任务)之间的分割，让每一个程序能自己运行而不收到其他程序的影响。同一个程序(任务)里也根据调用的指令的权限不同而进行隔离，避免指令的跨级误用
    3. 特权级就像是为根据指令执行时需要调用的系统资源的多少和深度来划分级别，以此来保证低级别的指令不会因为一些错误原因访问到高级指令所在地址区域从而跨级修改系统的数据。就像绕过了上面所说保护模式中为各个程序(任务)提供的保护屏障而导致OS出现预料之外的故障

第四章-进程与线程

(一)进程管理

1.进程概念

(1)进程基本概念

  • 并发环境运行的问题
    • 运行过程不确定
    • 过程无法再现
    • ——解决方案：对过程施加相互制约
  • 进程：描述和管理程序的运行过程
  • 定义：
    • 程序在某个数据集合上的一次运行活动
    • 数据集合：软硬件环境，多个进程共存/共享的环境
  • 特征：
    • 动态性：动态产生消亡
    • 并发性：与其他进程一起推进
    • 异步性：进程按照各自进度前行
      • 优点：不相关的进程之间不会互相干扰可以使整个系统的运行效率提升
      • 缺点：相关进程之间的通信会更为的繁琐且在编写进程时的调度策略问题处理会更加复杂
    • 独立性：系统扽配资源和调度CPU的单位
  • 进程与程序的区别：
    • 动态&静态：
      • 进程时动态的，程序是静态的
    • 暂存与长存：
      • 进程在内存驻留，程序保存在物理介质
      • 一个程序可能有多个进程
  • 进程的类型
    • 按使用资源的权限：
      • 系统进程：内核相关进程
      • 用户进程：运行于用户态的进程
    • 对CPU的依赖性
      • 偏CPU进程：计算型进程
      • 偏IO进程：偏重于IO的进程

(2)进程状态

  1. 状态的分类：
    1. 运行状态(Running):进程占有CPU在其上运行
    2. 就绪状态(Ready):具备运行条件但无CPU可运行
    3. 阻塞状态(Block)/等待状态(Wait):因为等待其他服务的完成或需要外设而无法运行
  2. 状态的变迁：依据一定的条件相互转化
    1. 就绪->运行：进程调度
    2. 运行->就绪：时间片到|被抢占
    3. 运行->阻塞：请求服务|等待信号
    4. 阻塞->就绪：服务完成|信号到达
    5. 特殊迁移能否发生？
      1. 阻塞->运行：CPU已经被分配无法直接进入CPU运行
      2. 就绪->阻塞：就绪态不会新申请外来资源，即所有需求资源在就绪态会一直处于满足要求状态
  3. 新建&终止：新建在就绪之前，终止在运行完成之后
  4. Linux进程状态：
    1. 可运行态：就绪&运行
    2. 阻塞~：
      1. 迁都阻塞：能被其他进程唤醒(等待信号类)
      2. 深度阻塞：不能被唤醒(等待CPU资源类)
    3. 僵死态：进程终止执行
    4. 挂起态：进程被挂起

(3)进程控制块PCB</br>

  1. 定义：
    1. 描述进程状态资源&与相关进程关系的数据结构
    2. PCB是进程的标志
    3. PCB随进程产生消亡
    4. 进程 = 程序 + PCB
  2. PCB基本成员：
  3. Linux的PCB：task_struct
  4. 进程的切换：
    1. 进程的上下文：进程运行环境-CPU环境
    2. 切换过程:
      1. 进入的进程的上下文出栈进入CPU
      2. 退出的进程的上下文进栈离开CPU

2.进程控制

(1)基本概念

  1. 概念：
    • 进程生存全期间，对其全部行为的控制
    • 四个典型的控制行为：
      1. 创建
        1. 功能
          • 创建一个具有指定ID的进程
        2. 参数配置：
        3. 创建过程：
          • 创建空白PCB
          • 获得并赋予进程标识符ID
          • 分配空间
          • 初始化PCB
          • 插入就绪队列
      2. 撤销
        1. 功能：
          • 撤销指定进程
          • 收回所占有资源，撤销ID
        2. 撤销的事件：
          • 正常结束
          • 异常结束
          • 外界干预
        3. 撤销的实现：
          • 检索PCB
          • 获取状态
          • 运行态立即终止，如果有子进程，递归撤销子进程
          • 释放占有资源
          • 将其PCB移除出队列
      3. 阻塞
        1. 功能：停止执行，变为阻塞
        2. 阻塞事件：
          • 请求系统服务
          • 启动某种操作
          • 新数据尚未到达
          • 无新工作可做
        3. 参数：
          • 阻塞原因
          • 不同原因构建不同的阻塞队列
        4. 实现：
          • 停止运行
          • 改变PCB状态
          • 插入阻塞队列
          • 转调度程序
      4. 唤醒
        1. 功能：唤醒处于阻塞队列的进程
        2. 唤醒世纪/事件
          • 满足系统服务
          • IO完成
    • 进程控制原语：
      • 若干指令构成的特定功能的函数
      • 具有原子性，不能中断该函数的运行
      • 例如进程创建/撤销/阻塞/唤醒原语

三 线程(Thread)

(一)基本概念

  1. CPU直接运行的实体
  2. 执行线程时，进程就不再是CPU的基本调度资源
  3. 一个进程可以创建多个线程且共享CPU进而实现并发

(二)执行

  • 使用函数CreateThread()
    • 功能：将函数做为线程来创建
    • 将需要执行的函数作为CreateThread的参数
  • 单线程程序：程序缺省只有一个线程(Main)，函数串行执行
  • 多线程程序：至少两个线程在程序中 其中一个必然为主线程

(三)线程典型使用场景

  1. 程序的多个功能需要并发运行
    • e.g：看电影
    • 视频解码，音频解码，网络接收
  2. 改善窗口交互
    • 需要进行后台计算
    • 文件的复制
  3. 多核CPU
  4. 协作完成同一件事情，需要共享的文件比较多

(四)使用线程的麻烦

  1. 程序难以调试
  2. 线程是否安全
  3. 并发的过程难以控制

四 临界区和锁

(一) 临界资源&临界区

  1. 临界区：区域内的操作只能够串行的运行，不能并发
  2. 临界资源：一次只允许一个进程独占访问的资源：比如共享变量?
  3. 同样的，临界区就是访问临界资源的代码块
  4. 访问特点：
    • 具有排他性
  5. 访问机制的四个原则：
    1. 临界区忙时，其他进程需要在临界区外等候
    2. 临界区空闲时，任何有权进程均可进入
    3. 进程的等待时间是由限制的
      • 临界区设置多大比较好？
